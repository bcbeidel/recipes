# RESOLVER.md

Routing table for filing new recipes and loading context before acting.
Machine-managed region regenerated by `/build:build-resolver`; human prose
outside markers is preserved.

<!-- resolver:begin -->
## Filing — where new content goes

| Content type | Location | Naming |
|---|---|---|
| Dinner main | `dinner/` | `<recipe-name>.md` (kebab-case) |
| Breakfast | `breakfast/` | `<recipe-name>.md` (kebab-case) |
| Lunch | `lunch/` | `<recipe-name>.md` (kebab-case) |
| Side dish | `sides/` | `<recipe-name>.md` (kebab-case) |
| Dessert | `desserts/` | `<recipe-name>.md` (kebab-case) |
| Appetizer or snack | `appetizers/` | `<recipe-name>.md` (kebab-case) |
| Basic / sauce / dough / component | `basics/` | `<recipe-name>.md` (kebab-case) |

## Context — what to look at before acting

| When doing… | Look in |
|---|---|
| Meal-planning a week | `_index.md`, `dinner/`, `sides/` |
| Planning a quick weeknight dinner | `_index.md` (Quick weeknights section), `dinner/`, `basics/` |
| Baking or making dessert | `desserts/`, `breakfast/` |
| Hosting / building an apps spread | `appetizers/`, `basics/` |
| Triaging a recipe with no `meal-type` frontmatter | `_index.md` (Missing data section), `basics/` |

## Out of scope

- `.git/` — version control internals
- `.obsidian/` — vault config managed by Obsidian
- `.claude/` — Claude Code project-local settings
- `attachments/` — image attachments (declared in `app.json`); not recipe content
<!-- resolver:end -->

## Notes

All recipe files carry frontmatter (`aliases:`, `tags:`, `meal-type:`,
`cuisine:`, `dietary:`, `cooking-time-minutes:`, `url:`, `rating:`).
`_index.md` uses Dataview queries against these fields.

Files in `basics/` that lack `meal-type:` are awaiting triage. Once a
meal-type is assigned, move the file to the matching folder.
